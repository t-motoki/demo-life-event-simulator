# バリデーションルール

作成日: 2026-04-07

---

## 原則

- エラーメッセージは **日本語** で出力する（FPが読める言葉で）
- バリデーションエラーが発生したら **処理を止めて** メッセージを表示する（無言で続行しない）
- 警告（続行可能）とエラー（続行不可）を区別する

---

## 確定済みバリデーション項目

| 項目 | 条件 | 区分 | メッセージ例 |
| ---- | ---- | ---- | ------------ |
| ローン年数 | 返済完了年が退職年齢を超える | エラー | `ローン返済が退職後まで続きます。年数を見直してください。` |
| 貯蓄残高 | マイナスになった | 警告 | `{年}年に貯蓄残高がマイナスになります。資金ショートの可能性があります。` |
| 住宅ローン控除 | 13年目以降も控除を適用しようとした | エラー（実装時に防止） | — （コードレベルで14年目以降は0にする） |
| 年齢 | 0以下または150以上 | エラー | `年齢の値が不正です。` |
| 年収 | マイナス | エラー | `年収にマイナス値は指定できません。` |

---

## エラー出力形式

```
エラー: {メッセージ}
処理を中断します。入力値を確認してください。
```

警告の場合:

```
警告: {メッセージ}
このまま続行しますか？ [y/N]
```

---

## 実装上の注意

- バリデーションは `src/input/` に集約する（ドメイン層に混在させない）
- 1つの入力値に複数エラーがある場合は **全件まとめて** 表示してから停止する
