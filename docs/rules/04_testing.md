# テスト記述ルール

作成日: 2026-04-07

---

## フレームワーク・ツール

- **pytest** を使う
- カバレッジ計測: `pytest --cov=src`

---

## ファイル命名

```
tests/
  domain/     test_<モジュール名>.py   例: test_housing_loan.py
  input/      test_<モジュール名>.py   例: test_validator.py
  output/     test_<モジュール名>.py   例: test_excel_writer.py
```

---

## テストの粒度

- **ドメイン層のテストが最重要**（計算ロジックは純粋関数なのでテストしやすい）
- 1テスト = 1ビジネスルール（「住宅ローン控除は13年目で0になる」など）
- テスト名は日本語 OK（`def test_住宅ローン控除は13年目で打ち切られる`）

---

## FP未確認事項のマーキング

計算モデルが未確定の箇所には必ず以下のコメントを付ける:

```python
# TODO: FPに確認（仮値）: 育休中の収入減率を仮で50%としている
```

テストにも対応するマーキングを入れる:

```python
@pytest.mark.xfail(reason="TODO: FPに確認（育休収入減モデル未確定）")
def test_育休中の収入は減額される():
    ...
```

---

## 境界値テストの必須項目

以下は必ずテストする:

| ケース | 理由 |
| ------ | ---- |
| 住宅ローン控除 13年目 → 0 | FP確認済みルール。回帰防止 |
| 貯蓄残高がマイナスになるケース | 資金ショート警告の動作確認 |
| ローン年数が退職後まで続くケース | バリデーションエラーの動作確認 |
| 年齢がマイナスのケース | 入力バリデーション |
