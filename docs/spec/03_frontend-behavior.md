# フロントエンド振る舞い仕様 — ライフイベント家計シミュレーター

作成日: 2026-04-14

---

## 目的

FP（ファイナンシャルプランナー）がクライアントと対面しながら、ブラウザ上でシナリオを入力し、
年次キャッシュフロー一覧を即座に確認できるようにする。

操作者は FP のみ。クライアント本人は操作しない。

---

## 画面構成

```
[クライアント情報セクション]
  本人情報 / 配偶者情報（任意）

[支出セクション]
  月間支出（生活費・保険・その他固定費）

[ライフイベントセクション]
  イベント一覧 ＋ 追加・編集・削除

[実行]
  「シミュレーション実行」ボタン

[結果セクション]
  年次キャッシュフロー一覧テーブル
```

すべてのセクションは 1 画面に縦並びで表示する。
タブ切り替えやページ遷移なく、FP がスクロールしながら入力できる。

---

## セクション別の振る舞い仕様

---

### 1. クライアント情報セクション

#### 目的

本人と配偶者の収入・退職条件を入力する。
FP が面談冒頭に聞き取った基本情報を素早く入力できる。

#### 入力項目（本人）

| 項目 | 型 | デフォルト値 | 説明 |
| ---- | -- | ------------ | ---- |
| 現在年齢 | 整数 | なし（必須） | |
| 税引後年収（円） | 整数 | なし（必須） | |
| 収入モデル | 選択 | 一定（flat） | 「一定」「昇給率指定」「定年後減額」の 3 択 |
| 昇給率（%） | 小数 | 0.0 | 収入モデルが「昇給率指定」のときのみ入力可 |
| 定年年齢 | 整数 | 65 | |
| 定年後年収（円） | 整数 | 0 | 収入モデルが「定年後減額」のときのみ入力可 |
| 年金受給開始年齢 | 整数 | 65 | |
| 年金額（円/年） | 整数 | 0 | FP 手入力 |

#### 入力項目（配偶者）

配偶者の入力エリアは「配偶者を追加」操作によって表示される。
表示後は本人と同じ項目を入力できる。
非表示にすると、配偶者のデータはリクエストから除外される（`null` 送信）。

#### 入力項目（シナリオ共通）

| 項目 | 型 | デフォルト値 | 説明 |
| ---- | -- | ------------ | ---- |
| 現在の貯蓄残高（円） | 整数 | なし（必須） | |
| シミュレーション終了年齢 | 整数 | 90 | 本人の年齢基準 |
| シミュレーション開始年（西暦） | 整数 | 現在年 | |

#### アクセプタンス基準

- 本人の現在年齢・年収・貯蓄残高が未入力のままシミュレーションを実行しようとすると、
  エラーメッセージを表示してリクエストを送信しない
- 収入モデルが「昇給率指定」のとき、昇給率の入力欄がアクティブになる
- 収入モデルが「定年後減額」のとき、定年後年収の入力欄がアクティブになる
- 収入モデルが「一定」のとき、昇給率・定年後年収の入力欄は非アクティブ（入力できない状態）になる
- 配偶者エリアを非表示にした状態で実行すると、リクエストの `spouse` は null になる

#### 境界値・異常系

| 条件 | 期待される振る舞い |
| ---- | ------------------ |
| 年齢に 0 または負数を入力して実行 | クライアント側でエラーメッセージを表示しリクエストを送信しない |
| 年齢に 101 以上を入力して実行 | クライアント側でエラーメッセージを表示しリクエストを送信しない |
| 貯蓄残高に負数を入力して実行 | クライアント側でエラーメッセージを表示しリクエストを送信しない |
| 終了年齢が現在年齢以下で実行 | クライアント側でエラーメッセージを表示しリクエストを送信しない |
| 年収に非数値を入力 | その欄にエラーを表示し、入力を受け付けない |
| 昇給率に 100% を超える値を入力して実行 | クライアント側でエラーメッセージを表示しリクエストを送信しない |

### 2. 支出セクション

#### 目的

毎月かかる固定支出を入力する。

#### 入力項目

| 項目 | 型 | デフォルト値 | 説明 |
| ---- | -- | ------------ | ---- |
| 月間生活費（円） | 整数 | なし（必須） | 住居費・食費などの合計 |
| 保険料（円/月） | 整数 | 0 | |
| その他固定費（円/月） | 整数 | 0 | |

#### アクセプタンス基準

- 月間生活費が未入力のままシミュレーションを実行しようとすると、
  エラーメッセージを表示してリクエストを送信しない
- 月間支出の合計（参考値）を表示する（編集の補助情報として）

#### 境界値・異常系

| 条件 | 期待される振る舞い |
| ---- | ------------------ |
| 月間生活費に 0 を入力して実行 | エラーなく実行できる（0 円の生活費は許容する） |
| 各項目に負数を入力して実行 | クライアント側でエラーメッセージを表示しリクエストを送信しない |

---

### 3. ライフイベントセクション

#### 目的

クライアントのライフイベントを追加・編集・削除する。
面談の会話の流れに応じて、FP がイベントを随時追加できる。

#### イベント一覧表示

追加済みのイベントをカード形式で一覧表示する。
各カードには、イベント種別・発生年・主要パラメータのサマリーを表示する。
イベントがゼロ件のときは「イベントがまだありません」のメッセージを表示する。

#### イベントの追加

「イベントを追加」ボタンを押すと、イベント種別を選ぶ。
選んだ種別に応じて、入力フォームが展開（またはモーダルで表示）される。
入力完了後に「追加」を押すと、一覧に追加される。

#### 対応イベント種別と入力項目

**結婚（marriage）**

| 項目 | 型 | デフォルト値 |
| ---- | -- | ------------ |
| 発生年（西暦） | 整数 | 必須 |
| 一時費用（円） | 整数 | 0 |

**出産（birth）**

| 項目 | 型 | デフォルト値 |
| ---- | -- | ------------ |
| 発生年（西暦） | 整数 | 必須 |
| 子ども人数 | 整数 | 1 |
| 本人育休中収入率（0〜1） | 小数 | 1.0（育休なし） |
| 本人育休期間（年） | 整数 | 0 |
| 配偶者育休中収入率（0〜1） | 小数 | 1.0（育休なし） |
| 配偶者育休期間（年） | 整数 | 0 |

**住宅購入（housing）**

| 項目 | 型 | デフォルト値 |
| ---- | -- | ------------ |
| 購入年（西暦） | 整数 | 必須 |
| 物件価格（円） | 整数 | 0 |
| 頭金（円） | 整数 | 0 |
| ローン年数 | 整数 | 35 |
| 金利（小数。例: 0.02 = 2%） | 小数 | 0.02 |
| 住宅ローン控除を使う | 真偽値 | true |

**教育（education）**

| 項目 | 型 | デフォルト値 |
| ---- | -- | ------------ |
| 発生年（西暦） | 整数 | 必須 |
| 対象の子の誕生年（西暦） | 整数 | 必須 |
| 幼稚園 | 選択（公立/私立） | 公立 |
| 小学校 | 選択（公立/私立） | 公立 |
| 中学校 | 選択（公立/私立） | 公立 |
| 高校 | 選択（公立/私立） | 公立 |
| 大学 | 選択（公立/私立） | 公立 |

**介護（care）**

| 項目 | 型 | デフォルト値 |
| ---- | -- | ------------ |
| 開始年（西暦） | 整数 | 必須 |
| 期間（年） | 整数 | 1 |
| 月額費用（円） | 整数 | 0 |

**その他支出（other_expense）**

| 項目 | 型 | デフォルト値 |
| ---- | -- | ------------ |
| 発生年（西暦） | 整数 | 必須 |
| 金額（円） | 整数 | 0 |
| 名称 | テキスト | 空文字 |

#### イベントの編集

一覧の各カードから編集できる。
編集フォームには現在の値を初期値として表示する。
「保存」を押すと一覧が更新される。

#### イベントの削除

一覧の各カードから削除できる。
削除は即座に反映される（確認ダイアログは任意）。

#### アクセプタンス基準

- 同じ種別のイベントを複数追加できる（例: 出産を 2 回）
- イベントを追加・削除した後、シミュレーションを実行すると最新のイベント一覧が反映される
- 追加したイベントは一覧に種別・年・主要パラメータのサマリーが表示される
- イベントがゼロ件のままシミュレーションを実行できる（イベントなしは有効な入力）

#### 境界値・異常系（住宅購入）

| 条件 | 期待される振る舞い |
| ---- | ------------------ |
| 頭金 >= 物件価格で追加 | クライアント側でエラーメッセージを表示し、イベントを追加しない |
| 金利に 0 未満または 1 超の値を入力 | クライアント側でエラーメッセージを表示し、イベントを追加しない |
| ローン年数に 1 未満または 51 以上を入力 | クライアント側でエラーメッセージを表示し、イベントを追加しない |

#### 境界値・異常系（出産）

| 条件 | 期待される振る舞い |
| ---- | ------------------ |
| 育休中収入率に 0 未満または 1 超の値を入力 | クライアント側でエラーメッセージを表示し、イベントを追加しない |
| 育休期間に負数を入力 | クライアント側でエラーメッセージを表示し、イベントを追加しない |

#### 境界値・異常系（共通）

| 条件 | 期待される振る舞い |
| ---- | ------------------ |
| 発生年を未入力で追加 | エラーメッセージを表示し、イベントを追加しない |

### 4. シミュレーション実行

#### 目的

入力内容を API に送信し、年次キャッシュフロー一覧を取得する。

#### 振る舞い

- 「シミュレーション実行」ボタンを押すと、入力内容の事前バリデーションを行う
- バリデーションが通ったら、`POST /simulate` にリクエストを送信する
- 送信中はローディング状態を表示し、ボタンを無効化する（二重送信防止）
- レスポンスが返ったら、結果セクションに一覧を表示する
- API がエラーを返したら、エラーメッセージを表示する

#### アクセプタンス基準

- 必須項目が未入力の状態ではリクエストを送信しない（クライアント側でブロック）
- 送信中はボタンが押せない
- API から結果が返ったら、結果セクションが自動的に表示領域に入る（スクロール）
- API から `detail` フィールドつきのエラーが返ったら、そのメッセージを表示する

#### 境界値・異常系

| 条件 | 期待される振る舞い |
| ---- | ------------------ |
| ネットワーク接続なしで実行 | 「サーバーに接続できません」のエラーメッセージを表示する |
| API がタイムアウト | 「応答がありませんでした。時間をおいて再試行してください」を表示する |
| API が HTTP 422 を返す | `detail` フィールドのメッセージを画面に表示する |
| API が HTTP 500 を返す | 「サーバーでエラーが発生しました」のメッセージを表示する |
| 送信中にユーザーが再度ボタンを押す | 2 回目のリクエストは送信されない（ボタンが無効化されているため） |

---

### 5. 結果セクション（年次キャッシュフロー一覧）

#### 目的

シミュレーション結果を FP とクライアントが一緒に確認できる形で表示する。
面談中に使うため、貯蓄残高がマイナスになった年を一目で判別できることが重要。

#### 表示内容

年次キャッシュフローをテーブル形式で表示する。

| 列 | 表示内容 |
| -- | -------- |
| 年 | 西暦（例: 2026） |
| 年齢 | 本人年齢 / 配偶者年齢（配偶者ありの場合） |
| 収入合計 | 円（整数） |
| 支出合計 | 円（整数） |
| 住宅ローン控除 | 円（整数） |
| 年間収支 | 円（正なら黒字・負なら赤字を色で区別） |
| 貯蓄残高 | 円（負なら赤字を色で区別） |
| イベント | その年のイベント名（" / " 区切りで複数表示） |

#### アクセプタンス基準

- テーブルはシミュレーション開始年から終了年まで全行を表示する
- 年間収支が負の行は視覚的に区別できる（例: 文字色を赤にする）
- 貯蓄残高が負の行は視覚的に区別できる（例: 背景色を薄い赤にする）
- 配偶者なしのシナリオでは「配偶者年齢」列を表示しない
- イベントが空の年は「イベント」列が空欄になる
- 結果テーブルはシミュレーション実行後にのみ表示される（初期状態では表示しない）
- 2 回目以降の実行では、前回の結果テーブルを新しい結果で上書きする

#### 境界値・異常系

| 条件 | 期待される振る舞い |
| ---- | ------------------ |
| 全年で貯蓄残高が黒字のシナリオ | 赤色表示なし・全行が通常表示 |
| 全年で貯蓄残高が赤字のシナリオ | 全行が赤色表示 |
| イベントが同一年に複数あるシナリオ | " / " 区切りで 1 セルに表示 |
| シミュレーション期間が 1 年のシナリオ | テーブルが 1 行で表示される |
| 配偶者あり → 配偶者を削除して再実行 | 配偶者年齢列が消え、新しい結果で更新される |

## 画面全体のアクセプタンス基準

1. FP がページを開いたとき、クライアント情報・支出・ライフイベントの入力エリアが最初から表示されている
2. ライフイベントがゼロ件でも、クライアント情報と支出を入力するだけでシミュレーションを実行できる
3. 入力エリアと結果テーブルは同一ページ内にあり、シミュレーション後にスクロールだけで確認できる
4. バックエンドが起動していない状態でページを開いたとき、接続エラーを適切に通知する（初期チェックは任意だが、実行時には必ず通知する）
5. ページをリロードすると入力内容はリセットされる（永続化なし）

---

## API 通信の仕様（フロントエンドから見た契約）

フロントエンドは以下の 2 エンドポイントのみ使用する。

| エンドポイント | 用途 | タイムアウト |
| -------------- | ---- | ------------ |
| `GET /health` | バックエンド起動確認 | 5 秒 |
| `POST /simulate` | シミュレーション実行 | 30 秒 |

リクエスト・レスポンスの形式は `docs/spec/02_api-behavior.md` を参照する。

---

## スコープ外（この仕様では定義しない）

- シナリオの保存・ロード
- 複数シナリオの比較表示
- Excel ダウンロード
- グラフ・チャート表示
- 認証・ログイン
- レスポンシブ対応（モバイル）
- 印刷レイアウト
