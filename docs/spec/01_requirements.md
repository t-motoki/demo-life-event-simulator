# 要件定義書 — ライフイベント家計シミュレーター

作成日: 2026-04-04
最終更新: 2026-04-04（計算モデル一部確定）

---

## 1. プロダクト概要

FP（ファイナンシャルプランナー）がクライアントとの面談で使う業務ツール。
クライアントのライフイベントと収入情報を入力すると、年次キャッシュフロー一覧をExcel形式で出力する。

**操作者**: FP（クライアント本人は操作しない）

---

## 2. ユーザーストーリー

- FPとして、クライアントの基本情報とライフイベントを入力したい。
- FPとして、年ごとの収入・支出・貯蓄残高を一覧で確認したい。
- FPとして、シミュレーション結果をExcelで出力し、面談資料として使いたい。
- FPとして、複数のシナリオ（楽観・悲観など）を比較したい。（将来要件）

---

## 3. 入力項目

### 3-1. クライアント基本情報

| 項目 | 内容 | 備考 |
|------|------|------|
| 現在年齢 | 本人・配偶者それぞれ | |
| シミュレーション終了年齢 | 例: 90歳 | |
| 現在の年収 | 本人・配偶者それぞれ（税込） | |
| 年収の推移モデル | 一定 / 昇給率指定 / 定年後減額 | ※ FP確認事項参照 |
| 現在の貯蓄残高 | | |

### 3-2. ライフイベント

以下のイベントをFPが任意に追加・設定できる。

| イベント | 設定項目 |
|----------|----------|
| 結婚 | 年（または年齢）、一時費用 |
| 出産 | 年（または年齢）、子ども人数 |
| 住宅購入 | 年、物件価格、頭金、ローン年数、金利、住宅ローン控除適用有無 |
| 教育 | 子どもごとに：幼稚園〜大学の区分（公立/私立）、留学の有無 |
| 老後 | 退職年齢、年金受給開始年齢 |
| 介護 | 年、期間、月額費用 |
| その他支出 | 年、金額、名称（自由入力） |

### 3-3. 支出の前提

| 項目 | 内容 |
|------|------|
| 現在の月間生活費 | 住居費・食費・その他の合計または内訳 |
| 保険料 | 月額 |
| その他固定費 | 月額 |

---

## 4. 出力仕様

### 4-1. Excelファイル構成

| シート名 | 内容 |
|----------|------|
| キャッシュフロー一覧 | 年次サマリー（メインシート） |
| 入力内容確認 | 入力したパラメータの一覧 |

### 4-2. キャッシュフロー一覧の列構成

| 列 | 内容 |
|----|------|
| 年 / 年齢 | 西暦と本人年齢（配偶者年齢） |
| 収入合計 | 本人給与 + 配偶者給与 + 年金 + その他 |
| 支出合計 | 生活費 + ローン返済 + イベント費用 + その他 |
| 住宅ローン控除 | 当年の控除額（年末ローン残高から算出） |
| 年間収支 | 収入合計 − 支出合計 + 住宅ローン控除 |
| 貯蓄残高 | 前年末残高 + 年間収支（運用益なし） |
| イベント | その年のライフイベント名（テキスト） |

### 4-3. ファイル出力

- フォーマット: `.xlsx`
- ファイル名: `cf_simulation_YYYYMMDD.xlsx`（日付は実行日）
- 出力先: 実行ディレクトリ（または指定パス）

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| 実行環境 | Python 3.x、Windows/Mac 両対応 |
| 依存ライブラリ | 最小限（openpyxl or xlsxwriter を想定） |
| 入力方法 | **YAML設定ファイル**（`scenario.yaml`）。今バージョンの制約。FPが直接編集する形式ではないため、面談前にエンジニアがシナリオを準備する運用を想定 |
| エラー処理 | 入力値の範囲外・矛盾を検出したらメッセージを表示して停止 |
| 処理速度 | 1シミュレーション実行は数秒以内 |

## 7. 計算モデルの前提

### 7-0. 確定済み事項

FPへの確認を経て決定した計算モデルの前提。

| 項目 | 決定内容 | 確定日 |
| ---- | -------- | ------ |
| インフレ率 | **0%固定**（生活費への加算なし） | 2026-04-04 |
| 運用利回り | **0%固定**（貯蓄残高への運用益加算なし） | 2026-04-04 |
| 住宅ローン控除 計算方法 | 年末ローン残高の **0.7%** を控除額として年間収支に加算する | 2026-04-07 |
| 住宅ローン控除 期間 | **13年**（13年目で打ち切り。14年目以降は控除額 = 0） | 2026-04-07 |
| 年金 | FPが手入力（自動試算なし） | 2026-04-07 |
| 教育費 | 幼稚園〜大学の公立/私立区分ごとの標準単価を使用 | 2026-04-07 |

---

### 7-1. インフレ・物価上昇 【確定: 0%固定】

~~生活費にインフレ率を毎年加算するか？~~
→ **加算しない。全期間0%固定。**

### 7-2. 貯蓄・資産運用 【確定: 0%固定】

~~貯蓄残高に運用利回りを加算するか？~~
→ **加算しない。全期間0%固定。**

### 7-3. 年金

- 公的年金（老齢基礎・厚生年金）を自動試算するか、FPが手入力するか？
- 自動試算する場合: 現行の年金額をそのまま使うか、将来の給付水準低下を見込むか？
- 受給開始年齢の繰り上げ・繰り下げに対応するか？

### 7-4. 住宅ローン

- 金利タイプ: 固定 / 変動 / フラット35 のどれを想定するか？
- 変動金利の場合: 金利変動シナリオをどう扱うか？
- 繰り上げ返済は考慮するか？

### 7-5. 教育費

- 教育費の標準単価は何を使うか？（文部科学省の調査データ等）
- 私立/公立の区分の粒度: 幼稚園・小・中・高・大学ごとに設定するか？
- 大学院・留学・習い事は別項目にするか？

### 7-6. 収入モデル

- 定年後の再雇用収入はどう扱うか？（比率指定 / 固定額 / 無視）
- 配偶者が育休・産休中の収入減をモデル化するか？
- 副業・不動産収入など給与以外の収入を項目として持つか？

### 7-7. 税・社会保険

- 所得税・住民税・社会保険料を計算に含めるか、税引後年収を入力とするか？
- 含める場合: 簡易計算（概算税率）か正確な税額計算か？

### 7-8. 死亡・保険

- 遺族年金・死亡保険金のシナリオを扱うか？
- 介護保険給付は費用と相殺して表示するか？
