# アクセプタンス基準 — ep4.4「根拠付き出力 — クライアントに渡せる形へ」

作成日: 2026-05-10

---

## 目的

FP がクライアントに渡せる成果物として、「根拠が書かれた PDF」を 1 クリックで手に入れられる状態にする。
現時点で存在する Excel・API の出力にコメント生成と PDF 出力を加え、面談のその場で渡せる資料を完成させる。

---

## 機能境界

| 機能 | 実装場所 |
| ---- | -------- |
| 機能1: 「前提条件・注釈」シートを Excel に追加 | `src/output/excel_writer.py`（`write_excel()` 拡張） |
| 機能2: FP コメントを Claude API で自動生成 | `src/output/comment_generator.py`（新規）+ `POST /generate-comment`（新規エンドポイント） |
| 機能3: PDF ダウンロード | `src/output/pdf_writer.py`（新規）+ `POST /download-pdf`（新規エンドポイント） |
| 機能4: フロントエンド ダウンロードボタン | React コンポーネント拡張 |

---

## 機能1: 「前提条件・注釈」シートの Excel 追加

### AC-1-1: シートが3枚目として追加されている

**Given** `write_excel()` を呼び出す
**When** 生成された `.xlsx` ファイルを開く
**Then** シートが「キャッシュフロー一覧」「入力内容確認」「前提条件・注釈」の順に 3 枚存在する

---

### AC-1-2: 作成日が実行日の日付になっている

**Given** 2026-05-10 に `write_excel()` を呼び出す
**When** 「前提条件・注釈」シートの作成日セルを読む
**Then** 値が `2026-05-10`（ISO 8601 形式）または `2026年5月10日`（日本語形式）である

---

### AC-1-3: FP 名の初期値が空文字列である

**Given** FP 名を指定せずに `write_excel()` を呼び出す
**When** 「前提条件・注釈」シートの FP 名セルを読む
**Then** セルの値が空文字列（または空白のみ）である

---

### AC-1-4: 収入モデル「flat」の説明文

**Given** `client.income_model == IncomeModel.FLAT`
**When** 「前提条件・注釈」シートの収入モデル説明欄を読む
**Then** 「収入一定」または「毎年同額の収入」を意味する日本語テキストが含まれている

---

### AC-1-5: 収入モデル「raise_rate」の説明文（昇給率を含む）

**Given** `client.income_model == IncomeModel.RAISE_RATE` かつ `client.raise_rate == 0.02`
**When** 「前提条件・注釈」シートの収入モデル説明欄を読む
**Then** 「昇給率 2.0%」（または同等の表現）が含まれている

---

### AC-1-6: 収入モデル「post_retirement」の説明文（定年後減額）

**Given** `client.income_model == IncomeModel.POST_RETIREMENT`
**When** 「前提条件・注釈」シートの収入モデル説明欄を読む
**Then** 「定年後」「減額」を意味する日本語テキストが含まれている

---

### AC-1-7: インフレ・運用利回りの説明文

**Given** どのシナリオでも（計算モデルの確定事項として 0% 固定）
**When** 「前提条件・注釈」シートの前提条件欄を読む
**Then** インフレ率 0%・運用利回り 0% の固定前提であることを示すテキストが含まれている

---

### AC-1-8: 住宅ローン金利の説明（HousingEvent あり）

**Given** `events` に `HousingEvent(interest_rate=0.015, ...)` が含まれる
**When** 「前提条件・注釈」シートを読む
**Then** 「住宅ローン金利 1.5%」（または同等の表現）が含まれている

---

### AC-1-9: 住宅ローン金利の説明（HousingEvent なし）

**Given** `events` に `HousingEvent` が含まれない
**When** 「前提条件・注釈」シートを読む
**Then** 住宅ローン金利に関する記載が存在しない

---

### AC-1-10: 貯蓄最低値の計算（最小 savings の特定）

**Given** キャッシュフロー一覧に複数の `CashFlowRow` が含まれ、`savings` の最小値が 2051年・50歳・1,200,000 円である
**When** 「前提条件・注釈」シートの注目ポイント欄を読む
**Then** 「2051年」「50歳」「1,200,000円」（または同等の表現）が含まれている

---

### AC-1-11: 貯蓄最低値が初期残高と同じ場合（単調増加シナリオ）

**Given** すべての `CashFlowRow.net >= 0` であり貯蓄残高が単調増加するシナリオ
**When** 「前提条件・注釈」シートを読む
**Then** 貯蓄最低値は最初の行の savings（初期残高 + 1年目 net）として記録されており、エラーは発生しない

---

### AC-1-12: 赤字期間の検出（net < 0 の年あり）

**Given** `rows` の中に `net < 0` の `CashFlowRow` が存在する（例: 2045年・2046年・2047年）
**When** 「前提条件・注釈」シートの注目ポイント欄を読む
**Then** 赤字期間として 2045〜2047年（または各年）が明示されている

---

### AC-1-13: 赤字期間なし（全年 net >= 0）

**Given** すべての `CashFlowRow.net >= 0`
**When** 「前提条件・注釈」シートを読む
**Then** 赤字期間に関する記載が存在しない、または「赤字期間なし」と明示されている

---

### AC-1-14: FP コメントが Klee One フォントで設定されている

**Given** `write_excel()` の引数として FP コメント文字列を渡す
**When** 生成された `.xlsx` の「前提条件・注釈」シートで FP コメントセルのフォント情報を読む
**Then** フォント名が `"Klee One"` に設定されている

---

### AC-1-15: FP コメントが空のとき欄は空のままである

**Given** FP コメントとして空文字列を渡す（コメント未生成の状態）
**When** 「前提条件・注釈」シートの FP コメント欄を読む
**Then** セルが空（または空白のみ）であり、エラーは発生しない

---

## 機能2: FP コメント自動生成（Claude API）

### AC-2-1: コメントが空でない

**Given** 有効な `Scenario` と `list[CashFlowRow]` を渡す
**When** コメント生成関数（または `POST /generate-comment`）を呼び出す
**Then** 返却された文字列が 1 文字以上である

---

### AC-2-2: コメントが 200 文字以内

**Given** 有効な `Scenario` と `list[CashFlowRow]` を渡す
**When** コメント生成関数を呼び出す
**Then** 返却された文字列の長さが 200 文字以下である

---

### AC-2-3: コメントが赤字年に言及している（赤字ありシナリオ）

**Given** `net < 0` の行が含まれるキャッシュフロー一覧（例: 2045年が赤字）
**When** コメント生成関数を呼び出す
**Then** 返却テキストに赤字年または赤字リスクを示す表現が含まれている（例:「赤字」「収支がマイナス」「資金不足」のいずれか）

---

### AC-2-4: コメントが貯蓄最低値に言及している

**Given** 有効なキャッシュフロー一覧（貯蓄最低値が特定できる）
**When** コメント生成関数を呼び出す
**Then** 返却テキストに貯蓄最低値の年齢または金額を示す表現が含まれている

---

### AC-2-5: 赤字なしシナリオでは赤字への言及がない

**Given** すべての `CashFlowRow.net >= 0` かつ `savings > 0`
**When** コメント生成関数を呼び出す
**Then** 返却テキストに「赤字」「マイナス」等の否定的な財務表現が含まれない

---

### AC-2-6: POST /generate-comment が 200 を返す

**Given** 有効なシミュレーション結果（シナリオ + キャッシュフロー一覧）を JSON で送信する
**When** `POST /generate-comment` を呼び出す
**Then** HTTP 200 が返り、レスポンス JSON に `comment` フィールドが含まれる

---

### AC-2-7: POST /generate-comment のレスポンス形式

**Given** 有効なリクエストを送信する
**When** `POST /generate-comment` が HTTP 200 を返す
**Then** レスポンスボディが `{"comment": "<文字列>"}` の形式であり、`comment` フィールドが文字列型である

---

### AC-2-8: Claude API 障害時は 503 を返す

**Given** Claude API が応答しない状態で `POST /generate-comment` を呼び出す
**When** リクエストを送信する
**Then** HTTP 503 が返り、`detail` フィールドにエラーメッセージが含まれる

---

## 機能3: PDF ダウンロード（POST /download-pdf）

### AC-3-1: Content-Type が application/pdf

**Given** 有効なシミュレーション結果を JSON で `POST /download-pdf` に送信する
**When** レスポンスを受け取る
**Then** HTTP 200 かつ `Content-Type: application/pdf` ヘッダーが付いている

---

### AC-3-2: レスポンスが有効な PDF バイナリである

**Given** 有効なリクエストを送信する
**When** レスポンスボディを受け取る
**Then** バイナリが PDF シグネチャ `%PDF-` で始まっている

---

### AC-3-3: キャッシュフロー一覧の全行が PDF に含まれている

**Given** 5 行のキャッシュフロー一覧を含むリクエストを送信する（例: 2026〜2030年の 5 行）
**When** 返却された PDF をテキスト抽出する
**Then** 2026、2027、2028、2029、2030 の各年（西暦）がすべてテキストとして含まれている

---

### AC-3-4: 前提条件・注釈の内容が PDF に含まれている

**Given** `client.income_model == IncomeModel.FLAT` のシナリオでリクエストを送信する
**When** 返却された PDF をテキスト抽出する
**Then** 収入モデルの説明テキスト（「収入一定」等）が含まれている

---

### AC-3-5: Klee One フォントが FP コメント欄に適用されている

**Given** FP コメントを含むリクエストを送信し、WeasyPrint で PDF 生成する
**When** PDF を生成する際に用いる HTML テンプレートを確認する
**Then** FP コメント欄に対応する CSS が `font-family: "Klee One"` を指定しており、かつ `/mnt/c/Windows/Fonts/KleeOne-Regular.ttf` が `@font-face` で参照されている

---

### AC-3-6: FP コメントが空のとき PDF に FP コメント欄が空のまま含まれる

**Given** `fp_comment` として空文字列を渡してリクエストを送信する
**When** 返却された PDF をテキスト抽出する
**Then** PDF にエラーはなく、FP コメント欄の見出しラベルは存在するが内容が空である

---

### AC-3-7: 配偶者なしシナリオでは配偶者年齢列が存在しない

**Given** `spouse == null` のシナリオでリクエストを送信する
**When** 返却された PDF をテキスト抽出する
**Then** 「配偶者年齢」「配偶者」を示す列ヘッダーが含まれない

---

### AC-3-8: 配偶者ありシナリオでは配偶者年齢列が含まれる

**Given** `spouse` が設定されたシナリオでリクエストを送信する
**When** 返却された PDF をテキスト抽出する
**Then** 「配偶者年齢」（または同等の列名）が含まれている

---

### AC-3-9: 貯蓄残高がマイナスの年は視覚的に区別できる

**Given** `savings < 0` の行を含むシナリオでリクエストを送信する
**When** 生成される HTML テンプレートを確認する
**Then** `savings < 0` の行に対して赤色背景またはそれに相当する CSS クラスが付与されている

---

### AC-3-10: 不正なリクエストボディには HTTP 422 を返す

**Given** 必須フィールドが欠落したリクエストを `POST /download-pdf` に送信する
**When** レスポンスを受け取る
**Then** HTTP 422 が返り、`detail` フィールドにエラーメッセージが含まれる

---

### AC-3-11: Content-Disposition ヘッダーでファイル名が指定されている

**Given** 有効なリクエストを送信する
**When** レスポンスヘッダーを確認する
**Then** `Content-Disposition: attachment; filename="cf_simulation_YYYYMMDD.pdf"` の形式でヘッダーが付いている（日付は実行日）

---

## 機能4: フロントエンド統合（ダウンロードボタン）

### AC-4-1: シミュレーション実行前はダウンロードボタンが表示されない

**Given** ページを開いた初期状態（シミュレーション未実行）
**When** 画面を確認する
**Then** PDF ダウンロードボタンが表示されていない

---

### AC-4-2: シミュレーション実行後にダウンロードボタンが表示される

**Given** 有効な入力を入れてシミュレーションを実行し、結果テーブルが表示された状態
**When** 結果セクションを確認する
**Then** 「PDF をダウンロード」（または同等のラベル）のボタンが表示されている

---

### AC-4-3: ダウンロードボタンクリックで PDF がダウンロードされる

**Given** シミュレーション結果が表示されている状態でダウンロードボタンをクリックする
**When** ブラウザの応答を確認する
**Then** `cf_simulation_YYYYMMDD.pdf` のファイル名でダウンロードが開始される

---

### AC-4-4: ダウンロード中はボタンが無効化される

**Given** ダウンロードボタンをクリックして `POST /download-pdf` のレスポンスを待っている状態
**When** ボタンの状態を確認する
**Then** ボタンが非活性（disabled）になっており、再クリックしてもリクエストが重複送信されない

---

### AC-4-5: API エラー時はエラーメッセージを表示する

**Given** バックエンドが HTTP 503 または HTTP 500 を返す状態でダウンロードボタンをクリックする
**When** レスポンスを受け取る
**Then** 「ダウンロードに失敗しました」（または同等のメッセージ）が画面に表示され、ボタンは再び活性化する

---

### AC-4-6: シミュレーション再実行後もダウンロードボタンが使える

**Given** 1 回目のシミュレーション実行後にダウンロードした状態
**When** 入力を変えて 2 回目のシミュレーションを実行する
**Then** ダウンロードボタンが表示されており、クリックすると最新のシミュレーション結果を反映した PDF がダウンロードされる

---

## スコープ外（この仕様では定義しない）

- Excel ファイルのダウンロード（既存の `write_excel()` はスタンドアロン利用のまま）
- FP コメントの手動編集 UI
- PDF の印刷プレビュー
- 複数ページにわたる PDF のページネーション制御
- 認証・認可
- Klee One フォントが OS にインストールされていない環境への対応（WSL2 + Windows 環境を前提とする）
