変更内容をコミットします。

以下の手順で進めてください：

1. `git status` と `git diff` で変更内容を確認する
2. `src/` 以下のコードファイルが含まれる場合のみ `python -m pytest tests/ -v` を実行する
   - ドキュメント・設定・スクリプトのみの変更ではスキップしてよい
   - 失敗があればコミット前に修正する
3. 変更を「意図・意味の単位」に分けて、それぞれ個別にコミットする
   - ドメイン変更（entities.py など）
   - インフラ・実装変更（reader, renderer など）
   - 設定・ドキュメント変更
   - スクリプト追加など
3. 各コミットメッセージは日本語で、以下の形式で書く：
   - 1行目: 変更内容の要点（動詞で始める）
   - 空行
   - 2行目以降: なぜ変更したか・どんな効果があるか（1〜3行）
   - 末尾: `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>`
4. コミットメッセージは HEREDOC 構文（`-m "$(cat <<'EOF' ... EOF)"`）で渡す
5. 全コミット後に `git log --oneline -5` で確認し、結果をユーザーに報告する

**注意事項:**

- `git add -A` や `git add .` は使わず、ファイルを個別に指定する
- `.env` や認証情報を含むファイルはコミットしない
- `--no-verify` は使わない
